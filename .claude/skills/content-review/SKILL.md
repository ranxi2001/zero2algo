# content-review：文章内容审查

本技能用于对 `zero2algo` 的 Markdown 文章进行结构化质量审查。

## 审查维度

### 1. Frontmatter 完整性

- 是否包含 `layout: default`
- 是否包含 `title`、`description`、`eyebrow`
- `eyebrow` 是否符合 `{模块名} / {NN}` 或 `Module {NN}` 格式

### 2. 问题优先

- 开篇是否说明这个问题为什么值得关注
- 是否引出真实算法岗位或面试中的常见困境
- 前几段是否能让读者知道这篇解决什么问题

### 3. 算法工程视角

- 是否讲清业务目标、数据、指标、模型和系统约束
- 是否说明方法在系统里扮演什么角色
- 是否区分离线指标和线上效果

### 4. 避免名词堆砌

- 是否只罗列模型名而缺少取舍说明
- 提到方法时是否说明适用场景和代价
- 是否有不加论证的“最佳实践”断言

### 5. 真实复杂度

- 是否主动提到采样偏差、数据泄漏、冷启动、延迟、成本、稳定性等问题
- 是否区分 Demo、离线实验和生产可用
- 是否说明失败场景或排查方向

### 6. 语言和结构

- 是否使用中文为主，保留必要英文术语
- 段落是否短而有信息密度
- 标题层级是否清晰
- 是否合理使用表格、代码块或 Mermaid 图

### 7. 文末导航

- 是否用“下一篇建议继续看：”收尾
- 链接是否使用相对路径和 `index.html`

## 输出格式

```markdown
## 审查报告：{文章标题}

整体结论：通过 / 需要改进 / 有明显问题

### 主要问题

1. {问题、位置、原因}
2. {问题、位置、原因}

### 修改建议

1. {具体怎么改}
2. {具体怎么改}

### 保留点

- {写得好的地方}
```
